模块划分

什么是模块?

在构建应用的时候,为了避免变量名、方法名的冲突.我们通常都会把它们放在不同的空间中. 例如: 
var Object = {
	m1:function(){
		//
	}
};
var OtherObject = {
	m1:function(){
		//
	}
};
Object.m1();
OtherObject.m1();
这样，就可以说Object,OtherObject两个对象就是两个 模块 在JavaScript中有AMD，CMD模块化的思想，这里只是简单的举个栗子.就不多介绍了.
那么在Angular中。模块的划分其实和JS的语法没太多关系.更多的是在业务逻辑上的划分(个人观点).在Angular1.X版本中.有controller,service,filter,directive,contant,value,factory.这些东西.都是挂载在模块module下的.例如:angular.module('app',[]).controller().service().这样一来,就根据模块来区分了不同业务逻辑之间的controller等的归属问题.不必再纠结是否重名等烦恼.
而且 模块之间还可以相互依赖,说直白点.就是类似java中的import 导入jar包.
